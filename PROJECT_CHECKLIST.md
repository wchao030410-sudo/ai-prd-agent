### 项目描述模板（简历用）

**项目名称：** AI PRD Agent - 智能产品需求文档生成工具

**项目描述：**
一款基于 AI 的产品需求文档生成工具，通过输入简单的产品想法，自动生成包含竞品分析、技术评估、RICE 优先级排序的专业级 PRD，实现 **10分钟完成原本需要2-3天的工作**。

**技术栈：**
- 前端：Next.js 16 + React 19 + TypeScript + Tailwind CSS
- 后端：Next.js API Routes + Prisma ORM
- 数据库：SQLite / Vercel Postgres
- AI：智谱 GLM-4.6 大语言模型

**核心贡献：**
1. 设计并实现 PRD 结构化数据模型，支持 8 大专业章节
2. 构建 Prompt Engineering 模板，确保 AI 输出质量稳定
3. 实现会话历史管理系统，支持版本追溯
4. 集成 RICE 评分方法论，实现数据驱动的功能优先级排序

**项目成果：**
- ✅ 完成核心功能开发（PRD 生成、历史管理、RICE 评分）
- ✅ 实现结构化 JSON 输出，解析成功率达 95%+

## 💡 面试准备

### 可能被问到的问题

**Q1: 为什么要做这个项目？**
> 作为产品经理，我发现撰写 PRD 是一个耗时且容易遗漏关键环节的工作。通过这个项目，我希望探索 AI 如何改变产品经理的工作范式，从"写文档"变成"审核决策"，实现真正的效率提升。

**Q2: 最大的技术挑战是什么？**
> 最大的挑战是确保 AI 输出的稳定性。智谱 AI 有时返回格式不统一，我通过以下方式解决：
> 1. 强制 JSON 输出格式
> 2. 添加健壮的解析和重试机制
> 3. 设计详细的 Prompt 模板

**Q3: 如果让你重新做，你会怎么改进？**
> 1. 使用 PostgreSQL 替代 SQLite，支持多用户场景
> 2. 添加流式响应，提升用户体验
> 3. 实现缓存机制，降低 API 成本
> 4. 添加更多 AI 模型选择（GPT-4、Claude）

**Q4: 这个项目的创新点在哪里？**
> 1. 专注于 PRD 垂直场景的深度优化
> 2. 集成 RICE 量化评分方法论
> 3. 自动化的竞品分析和技术评估
> 4. 结构化 JSON 输出，便于后续系统集成
